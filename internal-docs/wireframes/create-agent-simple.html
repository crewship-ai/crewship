Create Agent — Simple, single-screen MVP

Reaktion na "ten wizard je moc komplikovaný". Jeden dialog, vše vidíš najednou. Templaty jako shortcut chipy nahoře, ne jako separátní step. Persona/system_prompt vždy viditelná v textare. Advanced sbalené (~80 % uživatelů ho neotevře). Inspirace: OpenAI GPT Builder · Cursor custom modes · Linear new-issue · Notion AI blocks.

Parita s CLI & API. Tento mockup pokrývá všech 16 polí, která přijímá POST /api/v1/agents (a které ovládá crewship agent create). Zdrojem pravdy jsou enum hodnoty z prisma/schema.prisma:
  • AgentRole: AGENT · LEAD · COORDINATOR
  • CLIAdapter: CLAUDE_CODE · OPENCODE · CODEX_CLI · GEMINI_CLI
  • LLMProvider: OPENAI · ANTHROPIC · GOOGLE · OLLAMA
  • ToolProfile: MINIMAL · CODING · MESSAGING · FULL
  • lead_mode (jen když role = LEAD): active | passive

Pole temperature · max_tokens · webhook_secret · delegation_* jsou v DB schematu s defaulty, ale create endpoint je nepřijímá — nastavují se až po vytvoření přes PATCH na agent canvas. Wizard je proto neukazuje, aby se neslibovalo to, co create-time API neumí.

Kde to dělá konkurence stejně

  • OpenAI GPT Builder: jediný chat, generuje vše. Úplný extrém — žádný formulář.
  • Cursor custom modes: jeden dialog: jméno + system prompt textarea + model dropdown. Nic víc.
  • Linear new issue: jeden dialog. Title + popis + 6 popoverů (assignee, project, label, priority…). Templaty jako popover.
  • Anthropic Console (prompt generation): jeden textarea pro cíl, jeden tlačítko "generate", pak edit.
  • Notion AI custom block: jméno + instrukce. Hotovo.
  • Replit Agent: "Co chceš postavit?" → autopilot.

Crewship MVP equivalent: single dialog s template chips (≈ Cursor + Linear). Wizard je vzor pro onboarding nebo multi-tenant config, ne pro create-one-thing.
