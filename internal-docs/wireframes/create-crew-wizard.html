Notes for implementation

What to take from this mockup vs. what's flexible.

Hard requirements (these define the wizard)
  • 3 modes on Step 2: Browse templates (default), AI Suggest, Empty. Mode tabs at the top of Step 2 — switching mode does not reset Step 1 inputs.
  • 4 source tabs in template browser: built-in, mine, workspace, marketplace. Marketplace is future-only — render the tab disabled with "soon" badge.
  • Server-side search from the start — even built-in templates can grow. Endpoint: GET /api/v1/crew-templates?source=&q=&category=&limit=&cursor=. Debounce 200 ms. Virtualize once over ~30 items.
  • Preview shows the lineup, not the system prompt — the value of a crew template is the agent roster, not any single prompt. Show 3-5 rows: avatar + name + role title + LEAD/AGENT pill.
  • Edit lineup from preview = inline editor that lets the user remove rows or change role titles before applying. The crew gets created with the edited copy; the source template is unchanged.
  • AI Suggest is just step 2 mode, not a separate flow. It calls POST /api/v1/crew-ai-suggest (already exists in CLI as crewship crew suggest) and renders the result into the same lineup format. User can edit before applying.
  • Step 3 covers ALL CLI flags exposed in crewship crew create: --memory-mb, --cpus, --ttl, --network-mode, --allowed-domains. Today's UI lets you set zero of these — that's the main gap.
  • Skip semantics: Step 1 cannot skip. Step 2 skip = Empty crew (zero agents). Step 3 skip = template-derived defaults (or system-wide defaults for blank/AI).
Marketplace future-proofing (do this now, costs nothing)
  • Template ID format: crewmf_xxxxx for marketplace, crewtpl_xxxxx for workspace, crewb_xxxxx for built-in. Importing by ID becomes a single primitive.
  • Template JSON shape stays identical across sources — same fields whether built-in, workspace, or marketplace. Difference is metadata (origin, ★ rating, ↓ downloads, verified, last-update). Lets you "fork" any template into "mine" with one click.
  • Remember origin_template_id on the created crew. When marketplace template upgrades (v2.1), the crew settings page can offer "There's a new version — review changes". Without this, no upgrade story.
  • "Verified ✓" badge for built-ins and (later) marketplace-curated entries. Earned, not a self-toggle.
  • Stats block in preview (★ rating, ↓ uses) is rendered even for built-ins (just shows internal usage count) so the same component handles marketplace later.
Out of scope for v1 (good to know but skip for now)
  • Editing individual agents in template preview — for v1, "Edit lineup" only allows remove. Adding/customizing per-agent prompts happens in the per-agent canvas after the crew exists. AI Suggest mode is the exception (it must allow edit because each generated agent is unique).
  • Skills / credentials in wizard. Templates already declare which skills + credentials each agent needs; auto-assigned per CLAUDE.md rule. Manual assignment lives on the agent canvas.
  • Per-agent runtime overrides. Step 3 limits apply to the crew container (one per crew). Per-agent LLM/tool tweaks are on the agent canvas.
  • "Save as template" flow. Promoting an existing crew → workspace template is a separate task on the crew settings page.
  • A/B mode (try crew before commit). Could be a "Test in sandbox" button on Review later — would spin a temp crew for one task.
What's missing in today's dialog (the gap)
  • No icon picker — currently no UI; user can only set icon via CLI --icon. Yet CLAUDE.md mandates lucide-react names, not emoji. UI needs a curated grid of ~20 lucide icons + "More…" search.
  • No color picker — same story; UI must use the 8-palette IDs (blue, emerald, violet, amber, rose, cyan, lime, fuchsia), not free-form hex.
  • No description field — CLI has it, UI doesn't.
  • No container resource controls — memory, CPU, TTL all skipped in UI; default-only.
  • No network policy — Free vs Restricted + allowed-domains entirely missing from UI. This is the security knob most teams need first.
  • No AI Suggest — backend endpoint exists (POST /api/v1/crew-ai-suggest, called by crewship crew suggest --goal), but no UI surface.
  • Template browser is one flat list, no search/filter — fine for 6 templates, breaks at 30+. Marketplace will have hundreds.
