Crewship onboarding — 4 directions

Klikni na variantu pro funkční prokliké demo. Pak mi řekni která se ti líbí nejvíc.

Variant A

Classic wizard

Stepper nahoře, jeden krok najednou, Back/Next. Linear/Stripe esthetic. Bezpečná volba, nízké riziko.

  • + Familiar pattern, low cognitive load
  • + Easy to add/remove steps later
  • − "Seen it everywhere" feel
Recommended
Variant B

Template-first gallery

Hned na 1. screenu mapa crew templates s náhledem agentů. Výběr rozbalí inline workspace + adapter. Maximum wow.

  • + Fastest time-to-value (~60s)
  • + Templates as the hero
  • − 5 templates need polished art
Variant C

Single-page scroll reveal

Všechny sekce na jedné stránce. Vyplnění odemyká další. Bez Back/Next. Notion/Vercel feel.

  • + Power-user feel, dev-tool aesthetic
  • + Easy to scan/revise
  • − Less guidance for first-time users
Variant D

Split-screen w/ live preview

Levý panel form, pravý živý náhled crew. Při výběru "Software Dev" se vpravo ihned vykreslí 4 agent karty.

  • + Highest wow factor
  • + User vidí co buduje v real time
  • − Mobile breakpoint je drama
Mockup data: Software Dev / DevOps SRE / Marketing / Finance crew templates (existující seed_crew_templates.go). CLI adapters: Claude Code, Gemini, Codex, OpenCode, Cursor, Factory Droid (lib/cli-adapters.ts).
