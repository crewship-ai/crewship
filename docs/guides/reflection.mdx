# Reflection

The `reflection` package implements two multi-agent research patterns for self-improving agent outputs:

1. **Role-Based Reflection (MAR)** -- three specialist personas critique an output independently; an ensemble judge synthesizes the critiques into a single keep/revise/reject plan. Heterogeneous reviewers catch different failure modes than a single self-critique pass (published results show +6% on HumanEval and +3% on HotPotQA over single-agent reflexion).
2. **Evaluator-Optimizer Loop** -- a Generator produces an output, a Verifier tests it, failure feedback is folded back into the next Generate call. The loop terminates on first pass or after `maxIters` (default 5), at which point the caller is expected to escalate (e.g. to [Harbor Master](/guides/harbormaster) approval).

The package is provider-neutral -- LLM clients are local interfaces; callers wire adapters at the edges. The judge is reused from [Quartermaster](/guides/quartermaster) (`JudgeInterface`, `EnsembleJudge`) so bias-mitigation logic (rubric-shuffle) stays in one place.

## Personas

| Persona | What it surfaces |
|---|---|
| `logician` | Unjustified leaps, missing premises, contradictions between claims, invalid inferences. |
| `skeptic` | Challenges every factual assertion that lacks evidence. Treats the subject as a suspect, not a colleague. |
| `domain_expert` | Practitioner heuristics: patterns experienced people would avoid, red flags a beginner wouldn't notice, domain-specific correctness. |

The enum is deliberately small -- MAR ablation results show more personas dilute the signal and blow up latency/cost without adding coverage. The system prompts live in `personas.go`.

## Critique -> Synthesis

Each persona returns a `Critique`:

```go
type Critique struct {
    Persona     Persona
    Severity    CritiqueSeverity   // low, medium, high, critical
    Issues      []string
    Suggestions []string
    RawText     string             // populated when structured output fails to parse
}
```

`RawText` preservation is deliberate -- on parse failure the synthesizer still has signal and an operator can inspect the failure. Severity defaults to `low` on parse failure so one mangled critique cannot single-handedly trigger a reject.

The synthesizer (`quartermaster.EnsembleJudge` under the hood) reads all critiques and returns:

```go
type Synthesis struct {
    Keep       string           // prose describing what is already correct
    Revise     []RevisionPoint  // ordered, targeted edits with What + Why
    Reject     []string         // critique points discarded (contradiction, OOS, unsupported)
    Confidence float64
}
```

When the upstream ensemble reports high disagreement (stddev > 0.25) the `Confidence` is damped further so callers can treat low-confidence outputs as requiring human review (e.g. route to Harbor Master).

## Usage

```go
result, err := reflection.Reflect(ctx, deps, reflection.ReflectionRequest{
    Subject:  candidateOutput,
    Context:  taskDescription + relevantSources,
    Personas: nil, // nil -> AllPersonas
})
if err != nil { return err }

if result.Synthesis.Confidence < 0.6 || result.JudgeVerdict.HumanEscalate {
    // route to Harbor Master or an explicit human decision
    return escalate(ctx, result)
}

for _, rp := range result.Synthesis.Revise {
    log.Printf("revise: %s -- %s", rp.What, rp.Why)
}
```

`deps` bundles the persona LLM client, the quartermaster judge, and an optional journal emitter so critiques and syntheses land as `summary.generated` entries.

## Evaluator-Optimizer loop

```go
final, err := reflection.OptimizeLoop(ctx, deps, reflection.LoopInput{
    Task:      task,
    Generator: generatorClient,  // produces candidates given task + feedback
    Verifier:  verifierFn,       // tests the candidate; returns (pass, feedback)
    MaxIters:  5,
})
```

- Terminates on first pass or `MaxIters`.
- Failed iterations' feedback is folded into the next Generate prompt.
- On `MaxIters` without pass, `final.Escalated = true` -- route to Harbor Master.

Every iteration emits a `summary.generated` entry with the candidate + verifier feedback so the audit trail captures the convergence (or non-convergence) path.

## Gotchas

- **The package does not include an LLM client.** Callers pass adapters that implement the small `PersonaClient` / `Generator` interfaces. Keep adapters thin -- the reflection logic assumes the client is stateless and retry-safe.
- **Low confidence is not a failure.** If `Synthesis.Confidence` is 0.4 the synthesis itself is still valid; the caller should treat it as "ask a human" rather than "retry".
- **RawText on parse failure.** When you see `Severity=low` with `RawText` populated and empty `Issues`, the structured output parser failed -- common with smaller models. Swap the persona client for a stronger model or add explicit JSON-mode support in your adapter.
