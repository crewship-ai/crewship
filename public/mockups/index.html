Routine detail · style mockups

Five UI/UX patterns for the routine detail view. Same data (Eval: faithfulness RAG), different presentation. Pick one (or mix elements) — I'll implement.

Style 1
Linear / Notion
Clean hero, content-first, right rail with compact properties. Strong typography hierarchy. Best for "understand + edit".
Style 2
Stripe / Vercel
KPI tiles with sparklines + chart + cards in 2-column grid. Best for "operational overview".
Style 3
GitHub repo
Breadcrumb + about box + insights + README-style body + topics sidebar. Best for "discoverable doc".
Style 4
Datadog / Grafana
Time range bar + 6 KPI strip + multi-panel grid (line chart, p99 chart, pass-rate gauge, recent runs, spend breakdown). Metrics dense.
Style 5
n8n / Flowise
Visual DAG of routine steps (input → agent → output) on grid canvas + last-run trace bar + compact side panel.
Designed for desktop first; mobile responsive (single-column collapse) baked in for styles 1–4. Style 5's canvas needs touch consideration.
