## Known gaps (documented, not blockers for MVP)

- **Ollama not installed on dev VM** → memory consolidation skips each tick (`KEEPER_OLLAMA_URL` not set), Keeper gatekeeper stays disabled. Path is wired, falls back cleanly; real LLM-judge consolidation needs Ollama deployment.
- **No headless browser on dev VM** → item #9a and #13a–18a not verified with pixels; HTTP 200 from each route is the available evidence.
- **No Playwright suite** → browser tests are manual (one-shot screenshots, not regression-guarded).
- **Agent LLM calls bypass the middleware stack** (#4a) — Claude CLI tunnels through the sidecar CONNECT proxy direct to `api.anthropic.com`. The telemetry→paymaster→lookout→raw middleware composed in `internal/llm/middleware.go` only fires on server-side callers (Keeper, consolidate, quartermaster, summarizer). Agent cost tracking requires either (a) sidecar-side parsing of Claude CLI stream-json `usage` blocks to emit `cost.incurred` + `cost_ledger` rows, or (b) switching agent exec to direct `llm.Provider.Complete()` calls. Design decision, not a bug — but should be explicit in PRD.
- **Harbormaster timeout sweeper date parsing** (#12) — SQLite `datetime('now','+1h')` format vs RFC3339 expected by Go parser causes premature timeout fires. P2 follow-up.
- **Hooks/eval/presence/consolidate endpoints missing** — CLI stubs written, backend wiring is PR #205 scope.
- **mission_id not linked on agent exec entries** (#10a) — `exec.command` rows from agent containers don't populate `journal_entries.mission_id`, blocking per-mission checkpoint creation. Side bug.
